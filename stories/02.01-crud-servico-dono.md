## Story

**Como** dono do sistema,  
**quero** criar serviços (vinculando cliente, tipo, descrição, data e opcionais), alterar status, adicionar notas (interna ou visível ao cliente) e listar/filtrar/buscar serviços e ver detalhe com histórico completo,  
**para que** eu gerencie todos os atendimentos em um só lugar com rastreabilidade.

---

## Acceptance Criteria

1. Dono cria serviço informando cliente (existente ou novo), tipo (lista fixa), descrição, data abertura; opcionais: prazo, valor, endereço, contato. Sistema gera código único (ex.: SI-2026-0001) e status inicial ABERTO.
2. Dono altera status do serviço; sistema valida transições permitidas (matriz na arquitetura); rejeita transição inválida com 400.
3. Todo mudança de status gera registro em StatusHist (append-only); dono vê linha do tempo no detalhe do serviço.
4. Dono adiciona notas ao serviço com opção "Visível ao cliente"; notas aparecem no histórico (dono vê todas; cliente só as visíveis).
5. Dono lista serviços com filtros (status, período, cliente) e busca por código ou dados do cliente; abre detalhe com histórico completo.
6. Dono pode obter "Link para o cliente" (GET /api/servicos/:id/link-cliente) para enviar ao cliente.
7. Referência: ARQUITETURA-LOGICA-SITE-SI.md (entidades Serviço, StatusHist, Nota; transições 2.2; APIs 4.2.1).

## Tasks / Subtasks

- [ ] Backend: modelo Serviço (codigo único, id_cliente, tipo_servico, descricao, status_atual, data_abertura, etc.) e StatusHist, Nota.
- [ ] Backend: geração de codigo único (ex.: sequencial por ano).
- [ ] Backend: POST /api/servicos (criar serviço + primeiro StatusHist ABERTO).
- [ ] Backend: PATCH /api/servicos/:id/status (validar transição; atualizar Serviço + inserir StatusHist).
- [ ] Backend: POST /api/servicos/:id/notas (conteudo, visivel_cliente).
- [ ] Backend: GET /api/servicos (filtros, busca, paginação); GET /api/servicos/:id (detalhe + histórico + notas).
- [ ] Backend: GET /api/servicos/:id/link-cliente (gerar/retornar URL com token).
- [ ] Frontend: tela Novo serviço (formulário; seleção de cliente; lista tipos fixa).
- [ ] Frontend: lista de serviços (filtros, busca, cards/tabela); detalhe do serviço (linha do tempo, alterar status, nova nota, copiar link).
- [ ] Testes: criar serviço; transições válidas e inválidas; notas interna vs visível; listagem filtrada.
